# v0.2.0 - Refactor to Python Namespace Package

## Summary
Refactored Apollo-1 from a flat `src/` structure to a proper Python namespace package `apollo.apollo_1`, enabling future apollo project packages to share the `apollo` namespace.

## Changes Made

### New Files
- [pyproject.toml](file:///Volumes/Samsung970EVOPlus/dev-projects/apollo-1/pyproject.toml) - Modern Python package configuration
- [apollo/__init__.py](file:///Volumes/Samsung970EVOPlus/dev-projects/apollo-1/apollo/__init__.py) - Namespace package marker
- [apollo/apollo_1/__init__.py](file:///Volumes/Samsung970EVOPlus/dev-projects/apollo-1/apollo/apollo_1/__init__.py) - Package init with version and exports
- [apollo/apollo_1/agents/__init__.py](file:///Volumes/Samsung970EVOPlus/dev-projects/apollo-1/apollo/apollo_1/agents/__init__.py) - Agents subpackage

### Moved Files
| From | To |
|------|-----|
| `src/agents/srs_analyst.py` | `apollo/apollo_1/agents/srs_analyst.py` |
| `src/tools/__init__.py` | `apollo/apollo_1/tools/__init__.py` |
| `src/tools/knowledge_base.py` | `apollo/apollo_1/tools/knowledge_base.py` |
| `src/tools/redmine.py` | `apollo/apollo_1/tools/redmine.py` |

### Import Updates
Updated LangChain imports for v1.2+ compatibility:
- `langchain.tools` → `langchain_core.tools`
- `langchain.storage` → `langchain_core.stores`
- `langchain.retrievers` → `langchain_classic.retrievers`
